Project Documentation: ResilientDeep Prototype1. Project SummaryResilientDeep is a hardware-software prototype designed to detect "ShallowReals"—deepfakes that bypass standard security filters.Standard deepfake detectors rely on spotting digital compression artifacts to identify manipulated media. However, a "Critical Blind Spot" exists: adversaries can use heavy social media compression (like WhatsApp or TikTok) to destroy original artifacts , and then apply AI upscaling (like ESRGAN) to smooth the image out, effectively blinding standard detection models.This project solves that by implementing a Visibility Matrix to scan the frequency domain for invisible artifacts and a High-Frequency Enhancement network to physically restore fine details before passing the image to the classifier.2. Technical GlossaryHere are the core technical terms used in the codebase:ShallowReal: A highly compressed, then AI-upscaled deepfake designed to bypass artifact-based security filters.Social Media Compression: The act of drastically lowering video/image quality (e.g., JPEG compression or CRF 40) which destroys the high-frequency data that AI models normally use to spot fakes.Bicubic Interpolation / ESRGAN: Upscaling techniques. Bicubic is a standard mathematical way browsers resize images. ESRGAN (Enhanced Super-Resolution Generative Adversarial Networks) is an AI method that invents new pixels to make low-res images look sharp.Visibility Matrix Module: A custom neural network layer (implemented as a $1\times1$ Convolution in our code) that acts as a trainable frequency filter. It is designed to expose compression artifacts that are invisible to the naked eye.High-Frequency Enhancer: A small Convolutional Neural Network (CNN) block using residual connections. It takes the output of the Visibility Matrix and sharpens the edges digitally before the final classification.ResNet18 (The Backbone): A standard, highly robust deep learning image classifier. We cut off its original "head" and replaced it with a binary classifier (Real vs. Fake) to act as our final judge.$O(1)$ Time Complexity: In our data loader, this means the computer can fetch any image path from memory instantly (in constant time), rather than having to scan the hard drive every time it needs a new batch of data.3. Codebase File Directory & ExplanationsMaster Controllermain.pyRole: The entry point for the entire application. It acts as a command-line interface (CLI) router.Functionality: By parsing arguments (e.g., --mode train, --mode attack), it decides which part of the software pipeline to execute. It keeps the project modular so you don't have to run scripts manually from deep within the folder structure.Source Code (src/)A. Data Pipeline (src/data_pipeline/)dataset.pyRole: The PyTorch Data Loader.Functionality: It scans your dataset folders (Celeb-real and Celeb-synthesis) just once at startup and stores the file paths in a list. When the training loop asks for a batch of images, this script reads the images using OpenCV, converts them to RGB, and applies mathematical transformations (resizing to 224x224 and normalizing pixel values) so PyTorch can understand them.upscale.pyRole: The Attack Simulator (Phase 1).Functionality: It artificially creates the "Critical Blind Spot" data. It takes a pristine image, heavily compresses it to destroy its data, and then applies upscaling to smooth it out. We use this to generate the sample_attacked dataset to train our model on what an attack actually looks like.B. Core Modules (src/modules/)model.pyRole: The Neural Network Architecture.Functionality: Contains the three main deep learning classes:VisibilityMatrix: Learns to filter specific image frequencies.HighFreqEnhancer: Restores the lost edge details.ResilientDetector: The master class that wires the matrix, the enhancer, and the ResNet18 backbone together sequentially.C. Training Logic (src/training/)train.pyRole: The Execution Engine.Functionality: This script connects the dataset.py and the model.py. It pushes batches of images through the network (Forward Pass), calculates how wrong the network was (Loss), and adjusts the network's mathematical weights to be more accurate next time (Backward Pass / Backpropagation). It also actively saves the "brain" (best_model.pth) to your hard drive so you can use it later.D. Utilities (src/utils/)metrics.pyRole: The Grader.Functionality: Instead of just looking at the "Loss," this script calculates human-readable metrics. It compares the model's predictions against the actual answers and calculates the exact Accuracy (%) and the F1-Score (a specialized metric that balances false positives and false negatives).Hardware Interface (dashboard/)app.pyRole: The User Interface (UI).Functionality: Built with Streamlit, this creates a clean web interface simulating your final hardware product. It dynamically loads your fully trained best_model.pth weights into memory. When a user uploads a .jpg or .png, it converts the image, pushes it through the model, and outputs a real-time Probability Score and Verdict (AUTHENTIC vs. MANIPULATED).
